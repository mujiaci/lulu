## 抽卡页方案 B：星夜信笺

抽卡页不再使用单一渐变功能卡，而是使用一张竖屏星夜信笺背景：深蓝月夜、星轨、信封、蜡封和少量花瓣。图片不包含文字，Compose 负责叠加真实文案，避免字体和本地化问题。

页面层级从上到下：

1. 顶部返回与“限定卡池”小标签。
2. 中上部显示“星夜来信”标题、短句和当前卡池剩余信息。
3. 中部显示信封主视觉和稀有度色环，普通/紫/金/彩概率作为小型说明，不抢主视觉。
4. 中下部显示五类碎片的当前库存胶囊：抖音、剧场、游戏、视频、动漫。
5. 底部固定单抽、十连两个主操作，使用半透明深色底和金色/紫色强调边框，保证拇指区可达。

抽到紫色后，在结果卡片上明确写“抖音碎片”或“剧场碎片”；抽到金色后明确写“游戏碎片”或“视频碎片”。稀有度动画仍按彩色 > 金色 > 紫色优先级播放。

## 挂心状态与删除

辞海存储读取时把旧 `wake:07:30`、`schedule:叫醒` 等语义归一成同一个 canonical subject。应用启动时重建仍有效的 commitment 定时任务；刚过期但没有发送记录的任务进入一次补发队列，过期太久的任务显示“未送达”，不再显示“已经到点”并永久停留。

每张挂心卡右上角显示删除按钮。删除按 canonical subject 一并删除关联 concern、commitment 和对应定时任务，不留下孤立的过期卡。

## 拟人化状态

存储层和 UI 层都不能展示工具调用、API、planner、fallback、参数 JSON 等内部文本。旧 promise、contextText 和 actionPlan 摘要在读取时清洗；无法恢复自然语言时使用“我会在合适的时候再确认这件事”一类的人类可读兜底。

## 验收标准

- 安装最新构建后，三段动画与尾号映射一致。
- 抽卡页第一眼是星夜信笺视觉，而不是渐变概率卡。
- 辞海不会把无对应主动消息的旧记录标成“已经到点”。
- 删除挂心后，相关承诺和定时任务同时消失。
- 状态栏不出现工具名、参数、后台判断等技术句子。
