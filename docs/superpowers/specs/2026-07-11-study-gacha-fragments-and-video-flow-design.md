# 考研抽卡碎片与视频揭示流程设计

## 目标

调整考研计划中的轻运动偏好，重做娱乐碎片分类，并为紫色、金色、彩色三档抽卡提供稳定的全屏动画与视频碎片播放流程。

## 范围

- 第八套广播体操在需要安排轻运动时占 70% 的选择权重，其余轻运动共享剩余 30%。
- 紫色奖励拆为抖音碎片和剧场碎片，紫色内部各占 50%。
- 金色奖励拆为游戏碎片和视频碎片，金色内部各占 50%。
- 彩色奖励只产生动漫碎片。
- 移除旧通用紫色、通用金色字段、入口和余额，不迁移旧余额。
- 使用用户提供的三个 MP4，按金色、紫色、彩色分别加入应用资源。
- 不改变紫色、金色、彩色相对于普通奖励的总体稀有度，仅改变各档内部奖励分类。

## 数据模型

`StudyInventory` 使用用途明确的计数字段：

- `douyinFragments`
- `theaterFragments`
- `gameFragments`
- `videoFragments`
- `animeFragments`

旧的 `universalRareFragments`、`epicFragments`、`rainbowFragments` 和 `universalEpicFragments` 不再参与新逻辑。反序列化旧存档时允许旧字段被忽略，新字段按 0 初始化；不把旧余额转换到新系统。

每个 `StudyDrawResult` 需要携带明确的碎片种类，而不是只靠稀有度推断用途。抽卡结果入库、卡片标题、库存展示和兑换入口都依据该种类处理。

## 抽卡概率

保持当前档位概率：

- 紫色总概率维持现状，命中后抖音 / 剧场各 50%。
- 金色总概率维持现状，命中后游戏 / 视频各 50%。
- 彩色总概率维持现状，固定为动漫碎片。

随机分配必须由可注入的 `Random` 完成，以便单元测试验证边界和近似均分行为。

## 开场动画选择

单抽和十连使用同一优先级，整批只播放一次最高稀有度动画：

1. 批次包含彩色：播放彩色动画。
2. 否则包含金色：播放金色动画。
3. 否则包含紫色：播放紫色动画。
4. 只有普通奖励：直接进入卡片揭示。

三个动画均全屏播放。动画完成后释放视频播放器，并从该动画提取最后一帧作为全屏静态背景，直到本批抽卡结束。

## 视频碎片分配

抽到视频碎片时立即为该条揭示结果预留一个视频：

1. 从可见且未解锁的视频中随机选择，优先未解锁集合。
2. 若未解锁集合为空，从已解锁可见视频中随机选择。
3. 若没有任何可见视频，该碎片仍正常入库，揭示时显示卡片但不尝试播放奖励视频。

同一十连包含多枚视频碎片时，每枚都形成独立的待播放项。允许随机到不同视频；如果可用视频不足，可重复选择已解锁视频。

## 全屏播放状态机

流程分为互斥阶段：

1. `OpeningVideo`：唯一播放器全屏播放所选档位动画。
2. `Card`：播放器已释放，显示抽卡动画末帧背景和当前卡片。
3. `RewardVideo`：当前卡片是视频碎片时，用唯一播放器全屏覆盖背景播放对应视频。
4. `Card`：奖励视频结束后释放播放器，恢复末帧背景。
5. `Summary`：所有卡片及强制奖励视频处理完毕后显示汇总。

任何时刻最多存在一个活动的 `VideoView`。冻结背景只使用 `Bitmap`，不保留暂停的抽卡播放器，避免两个 Surface 或硬件解码器互相抢占。

## 点击与跳过行为

- 正常翻卡：到达视频碎片卡片时，先全屏播放其对应视频，完成后回到该卡片，再允许继续。
- 跳过开场动画：停止开场播放器、取得可用末帧并进入卡片阶段，不关闭整个抽卡流程。
- “跳过全部”：跳过普通卡片动画，但不能跳过视频碎片奖励；按抽卡结果顺序依次播放所有尚未播放的视频，全部完成后进入汇总。
- 关闭奖励视频：视为该奖励视频已处理，释放播放器并继续下一待播放视频或回到卡片。
- 视频加载失败：标记当前视频已处理、释放播放器并继续流程，不能卡死抽卡弹窗。

## UI 与资源

- 三个用户视频复制到 `app/src/main/res/raw/`，使用稳定的资源名：
  - `star_wish_epic_draw.mp4`：金色（替换现有资源）
  - `star_wish_rare_draw.mp4`：紫色（新增）
  - `star_wish_rainbow_draw.mp4`：彩色（替换现有资源）
- 抽卡动画与奖励视频都采用沉浸式全屏展示。
- 卡片、库存和兑换按钮使用现有 Material 3 组件与主题色，不在叶子组件内加入业务状态转换。
- 库存展示改为抖音、剧场、游戏、视频、动漫五类，不再展示“通用紫色/通用金色”。

## 广播体操权重

计划生成提示明确表达轻运动的加权偏好：当日需要轻运动时，第八套广播体操约占 70%，手势舞、散步、原地踏步、开窗站立和关节活动等共享约 30%。天气、身体状态和恢复需求仍可覆盖该偏好。

## 测试

- 抽卡规则测试覆盖紫色与金色内部 50/50 分流及彩色固定动漫碎片。
- 库存测试覆盖五种碎片分别入库和分别消费。
- 存档测试覆盖旧通用字段被忽略且新字段从 0 开始。
- 揭示流程测试覆盖紫色开场、彩色 > 金色 > 紫色优先级、单抽和十连。
- 状态机测试覆盖开场末帧背景、单播放器互斥、逐张视频播放、跳过全部仍播放所有视频、加载失败继续。
- 计划提示测试断言广播体操 70% 偏好被写入生成约束。

## 验收标准

- 三档非普通稀有度均播放正确的全屏开场动画。
- 十连只播放最高稀有度动画一次。
- 开场完成后末帧持续作为背景，奖励视频可以正常全屏播放并返回该背景。
- 十连中的所有视频碎片在正常翻卡和跳过全部两条路径下都不会漏播。
- 应用中不再存在可自由选择用途的通用紫色或通用金色碎片。
- 广播体操轻运动权重为 70%。
