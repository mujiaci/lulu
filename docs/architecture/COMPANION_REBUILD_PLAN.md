## 不可更改的产品规则

1. 记忆按用户在角色页面设置的批次大小 `B` 和最近保护数量 `T` 处理；`20` 与 `10` 只是新角色默认值，不是固定规则：
   - 第一批 `1～B`
   - 第二批 `B+1～2B`
   - 第三批 `2B+1～3B`
   - 例如用户设置 `B=20` 时才是 `1～20`、`21～40`、`41～60`。
   - 不允许拆分正式批次，也不允许将不同标准批次的未处理残片拼成一批。
2. checkpoint 只在“成功提取到记忆”或“模型明确判断无可记录内容”后推进；调用、解析或校验失败不得推进。
3. 开启 `globalApply` 的世界书必须完整进入所有角色、所有角色行为生成场景，不能被相关性筛选过滤。
4. 全局世界书只能从纯本地计算或非角色机械任务中省略；不得因 token 预算静默裁剪。
5. 普通回复、主动回复、通话、日记、游戏等角色出口必须遵循统一人设与世界观约束。
6. 正文、简短心声、状态栏和动作优先由同一次生成产生；不进行每轮第二次完整上下文的“心声模型”调用。
7. 保留固定感知，但本地心跳不调用模型；只有事件或值得深度判断的变化才调用意图/规划模型。
8. 不使用固定每日主动联系次数，也不把用户忙碌解释为“不爱角色”或作为惩罚依据。
9. 停止用多巴胺、血清素、皮质醇等神经递质数值模拟爱意或进入提示词/主动决策。
10. 角色不能声称完成数据库中没有真实执行记录的日记、收藏、游戏、计划或承诺。

### P1：用户可配置的连续记忆批次

- [x] 批次大小读取角色级 `memoryExtractionInterval`；默认 20，用户可在角色页面调整，0 表示关闭自动提取。
- [x] 保护区默认 10 条，并支持角色级调整；自动提取、最近批、继续历史、完整重建和记忆页均读取同一角色设置。
- [x] 先按用户设置值切分标准连续批次，再读取 checkpoint；不跨批次拼接历史空洞。
- [x] 建立持久化独立批次状态：`PENDING`、`PROCESSING`、`SUCCESS_WITH_MEMORIES`、`SUCCESS_EMPTY`、`FAILED_RETRYABLE`、`FAILED_MANUAL_REVIEW`、`INVALIDATED`。
- [x] 保存会话、序号范围、消息 ID、时间范围、尝试次数、脱敏错误类别、模型、提取版本和生成记忆 ID；数据库由 v28 安全迁移至 v29。
- [x] 将过渡值 `branch_id=selected` 替换为按批次终点计算的稳定分支指纹；追加新消息不会改变旧批次身份，不同选中分支互不冒领成功。
- [x] 区分“无记忆”与“提取失败”，落实安全 checkpoint；确定性结果不能掩盖语义失败。
- [x] 网络、连接、限流和服务暂不可用等临时失败自动重试；JSON 只允许一次保守格式修复；累计三次失败进入人工修复。
- [x] 确定性规则与语义提取分轨，语义失败时不得被确定性结果掩盖。
- [x] 编辑、删除或切换旧消息分支时精确标记相关批次为 `INVALIDATED`；删除节点导致边界移动时向后失效，旧生成记忆退役并等待完整窗口重建。
- [x] 修复历史空洞：按固定边界重建整批，并按 `sourceMessageIds` 去重。
- [x] 区分消息时间、事件实际时间、推测时间、记忆创建和更新时间；数据库由 v29 安全迁移至 v30。
- [x] 将 `dailySummary` 改为带 `sourceMemoryIds` 的派生摘要；与其覆盖的原子记忆互斥召回。
- [x] 记忆页展示成功点、下一批起点、稳定区终点、剩余条数、失败/空结果/失效批次及手动重试入口。

### P2：统一上下文与全局世界书

- [ ] 建立唯一 `CompanionContextEnvelope` 装配器。
- [ ] 普通聊天、主动消息、通话、日记、便签、游戏、关系判断和状态更新统一走装配器。
- [ ] 固定层包含系统边界、全局世界书、角色核心与角色专属常驻世界书。
- [ ] 条件触发世界书才允许按关键词/地点/人物/事件召回。
- [ ] 最近聊天采用固定滑动窗口，旧原文由滚动摘要替换。
- [ ] 记忆 Top-K、去重、多样化召回；日总结与原子记忆互斥。
- [ ] 承诺、关注和状态只注入当前相关的结构化短字段。
- [ ] 各场景拥有独立预算；优先缩减旧聊天和低相关内容，不裁剪全局世界书。
- [ ] 超预算且无法安全缩减时向用户明确提示。
- [ ] token 统计至少分为：角色核心、全局世界书、角色世界书、最近消息、滚动摘要、记忆、关系/状态、承诺/关注和其他提示词。
- [ ] 长聊后输入应在范围内波动，不再随历史永久线性增长。

### P3：主动代理与统一输出

- [ ] 三层触发：事件触发、本地心跳、深度感知。
- [ ] 本地心跳只更新时间、检查到期事项和累计离线时长。
- [ ] 没有意图模型时，本地决策器仍能处理承诺、关注、忙碌、离线、活动、未反馈消息和未发送计划。
- [ ] 意图判断与内容生成均完整注入全局世界书。
- [ ] 统一输出：正文、简短心声、状态栏、可选动作、状态变化、后续关注和再次联系计划。
- [ ] 区分 `lastUserActivityAt`、`lastUserReplyAt`、`lastOrdinaryAssistantAt`、`lastOutboundAt`、打开/回复时间、结果和生活锚点更新时间。
- [ ] 角色主动发消息不能重置用户离线计时。
- [ ] 主动联系生命周期覆盖生成、发送、送达、打开、暂未回复、后来回复、用户忙、转话题、拒接和提醒完成。
- [ ] 反馈影响下一次时间、方式、长度和是否追问，但不产生惩罚性反应。

### P4：关系、承诺与可解释状态

- [ ] 根据完整角色卡一次性初始化关系身份、认识时长、共同经历、阶段、安全感、依恋表达、印象、边界和潜在矛盾。
- [ ] 实际互动以阶段、证据、原因、置信度和变化时间发展。
- [ ] 页面主要展示关系说明、互动习惯、边界、未解决矛盾与证据，不用单个数值代替关系。
- [ ] 一句话中的多个承诺自动拆分；“监督起床、睡觉和学习”必须形成三项。
- [ ] 每项承诺保存承诺者、对象、责任、时间/频率、条件、执行方式、状态、最近执行及完整历史。
- [ ] 旧神经递质字段先兼容读取，但停止进入提示词和决策；迁移稳定后再删除。
- [ ] 新状态包含情绪及原因、精力、平静/紧张、安全感、防御、开放度、社交意愿、注意焦点、忙碌度、未解决关注和未完成想法。
- [ ] 关键状态带原因、证据、更新时间、有效期和置信度。
- [ ] 生活锚点记录活动、情境/数字空间、联系便利度、开始/预计结束、来源、有效期和置信度。
- [ ] 新明确事实覆盖旧锚点；关注持续到事件结束、收到结果或置信度自然衰减。

### P5：收藏与数字生活

- [ ] 角色可自主收藏有情感意义的消息，不使用固定概率/轮次。
- [ ] 收藏保存消息 ID、角色、原因、当时感受、时间及自主/手动来源。
- [ ] 首批真实活动：私人日记、未发送便签、整理收藏、回顾经历、整理关注、玩小游戏、看回放、共同计划、承诺复盘、状态与待续念头整理。
- [ ] 每项活动定义触发条件、冷却、耗时、是否用模型、真实存储、记忆写入和后续提及规则。
- [ ] 活动选择器、执行器和存储层全部注册后才允许角色声称完成。
